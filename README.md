# XUI Reseller Panel - Hardened JSON Edition

Production-oriented PHP reseller panel for 3x-ui nodes with **admin + reseller roles**, **GB credit accounting**, **fixed-duration subscriptions**, **one customer = one selected permitted server/inbound**, and **JSON storage with atomic writes**.

## Included
- Admin and reseller authentication
- JSON storage engine using per-record files, `flock()` locking, temp-write + rename
- Multiple 3x-ui nodes
- Inbound template import from 3x-ui
- Per-reseller allowed template assignment
- Reseller credit in **GB**
- Fixed reseller duration in **days**
- Customer creation on **one selected server/inbound template only**
- Public subscription route: `/user/<subscription_key>`
- Export route: `/user/<subscription_key>/export`
- Customer traffic sync from 3x-ui
- Remote enable/disable/delete reconciliation
- Ticketing between admin and reseller
- Vanilla CSS/JS only, no external libraries

## Server requirements
- PHP 5.6+
- cURL enabled
- OpenSSL enabled (recommended)
- `mod_rewrite` on Apache or equivalent rewrite rules on Nginx/Caddy

## Deploy
### Apache
1. Upload the project.
2. Point the virtual host to the project root **or** to `public/`.
3. Ensure `.htaccess` is allowed.
4. Make `storage/` writable by PHP.
5. Open `/install`.

### Nginx
Use `public/` as web root, or route all requests to `public/index.php` while serving `/assets/` from `public/assets/`.

## First setup
1. Finish `/install`.
2. Add a 3x-ui server under **Servers**.
3. Test the node.
4. Import inbounds.
5. Review imported templates and adjust labels if needed.
6. Create reseller accounts.
7. Assign permitted templates to each reseller.
8. Credit the reseller in **GB**.
9. Reseller creates users using one selected allowed template.

## Important behavior
- Credit is deducted only after a customer is saved successfully.
- Customer duration is fixed by the reseller profile.
- Subscription links are generated by this panel, not directly by 3x-ui.
- Template import stores raw inbound JSON to improve export fidelity.
- TLS verification for nodes is enabled by default; only disable it for trusted internal/self-signed environments.

## Writable paths
- `storage/`
- subdirectories under `storage/` are created automatically if missing

## Notes
- 3x-ui API behavior can still vary slightly by release; this panel includes fallback routes and response handling, but live testing against your exact node is still recommended.
- The panel is SQL-free by design, using JSON files only.


## Local QR

QR images for the public /get page are now generated locally by the panel via the bundled QR helper and no longer depend on an external QR website.


## Web exposure hardening

This package now blocks direct URL access to `storage/`, `app/`, `config.php`, hidden files, and common sensitive file types when deployed under Apache or LiteSpeed with `.htaccess` enabled.

Recommended deployment remains:
- best: web root points to `public/` only
- acceptable: project root is web-accessible with the included `.htaccess` rules active

If you deploy on Nginx, you must add equivalent deny rules in the server config because `.htaccess` will not be used there.
